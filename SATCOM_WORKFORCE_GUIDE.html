Section 03

Authentication & Login

Secure JWT-based authentication with OTP device verification, token rotation, and multi-device logout support.

🔒

Login Flow

How users access the platform

All Roles

Users authenticate with their company email and password. The system issues a JWT access token and a refresh token. New device logins trigger OTP verification via email for additional security.

  1. Navigate to the login page — Open the web app at http://localhost:3000 or launch the mobile app. You will be redirected to the login screen.
  2. Enter credentials — Type your company email and password, then click "Sign In".
  3. Device verification (if new device) — A 6-digit OTP is sent to your email. Enter the code to verify.
  4. Dashboard redirect — You are automatically routed to your role-specific dashboard.
Web: /login Mobile: /login API: POST /auth/login
🔑

Password Recovery

Self-service password reset

Users can reset their password without admin intervention. A reset link is sent to their registered email address.

  1. Click "Forgot Password" on the login page.
  2. Enter your email — The system sends a reset link.
  3. Open the email and click the reset link.
  4. Set a new password and confirm. You can then log in normally.
Web: /forgot-password Web: /reset-password

Section 05

Attendance & Check-In

GPS-enabled attendance tracking with work mode selection, break management, and multi-session support throughout the day.

📍

Check-In / Check-Out

Core daily attendance recording

Manager Employee

Employees check in at the start of their workday and check out when done. The system captures GPS coordinates, device info, and work mode. Multiple check-in/check-out sessions per day are supported (e.g., if an employee leaves for lunch and returns).

  1. Open the Dashboard — The check-in card is prominently displayed at the top.
  2. Select Work Mode — Choose from Office, Remote, Customer Site, Field Visit, or Travel.
  3. Tap "Check In" — GPS location is captured automatically. If geofencing is enabled, location is validated against office boundaries.
  4. Work Timer Starts — A live counter shows elapsed work time. You can select your current project/task.
  5. Take Breaks — Use "Start Break" or "Start Lunch" buttons. Break duration is tracked separately.
  6. Tap "Check Out" — Day summary is generated showing total work time, breaks, and overtime.
Work Modes: Office (on-site), Remote (WFH), CustomerSite (client location), FieldVisit (outdoor assignment), Travel (in transit).
Web: /dashboard Mobile: /(tabs)/home API: POST /attendance/check-in API: POST /attendance/check-out

Timesheets & Time Tracking

🕑

Timesheet Entry

Log time against projects and tasks

Manager Employee

Employees log their work time against specific projects and tasks. Entries include duration in minutes, notes describing the work performed, and optional file attachments (screenshots, documents). Entries go through a Draft → Submitted → Approved workflow.

  1. Navigate to Timesheets — Click "Timesheets" in the sidebar or dashboard quick actions.
  2. Click "New Entry" — A form opens with project/task selector, date, and duration fields.
  3. Select Project & Task — Choose from assigned projects. Tasks are filtered per project.
  4. Enter Time & Notes — Enter duration (in minutes), add a description of work done.
  5. Attach Files (optional) — Upload supporting documents or screenshots.
  6. Submit — Entry moves from Draft to Submitted and awaits approval.
StatusesDraft, Submitted, Approved, Rejected

The system automatically detects anomalous patterns in attendance, timesheets, and geofence data. Each anomaly is categorized by type and severity, and follows an Open → Acknowledged → Resolved workflow.

Geofence & Office Locations

Define office boundaries with GPS coordinates and radius to verify employee check-ins occur within approved areas.

🌏

Office Location Management

Add and configure office geofence zones

SuperAdmin

SuperAdmins can add office locations with GPS coordinates and define a geofence radius. When employees check in, their GPS position is compared against these boundaries. Locations can be activated or deactivated as needed.

  1. Go to Locations — Navigate to Admin → Office Locations.
  2. Click "Add Location" — A modal form opens.
  3. Enter Details — Name (e.g., "HQ Mumbai"), full address, latitude/longitude, and radius in meters.
  4. Save — The location appears in the table and is immediately active for geofence validation.
  5. Edit or Deactivate — Use the action menu on any location row to modify or disable it.
Geofence Policy: Enable/disable geofencing globally, require office-only check-ins, or allow bypass with reason — all configurable in System Settings.
Web: /admin/locations API: POST /locations API: GET /locations

User Management

Full employee lifecycle management including onboarding, role assignment, profile management, and status control.

👥

User CRUD Operations

Create, edit, and manage employee accounts

SuperAdmin HR

SuperAdmins can create new user accounts, assign roles, set department and manager assignments, and manage account status. HR users can view and search the user list. The system supports search, filtering by role/status, and pagination for large organizations.

Important: User creation is done via POST /api/v1/users (not /auth/register). Only SuperAdmin can create users. The request body requires a nested profile object with employee details.

Add New User (Step by Step)

  1. Navigate to User Management — Go to Admin → Users. Only SuperAdmin sees the "Add User" button.
  2. Enter Employee Code — A unique identifier like EMP001 (required, must be unique across the system).
  3. Set Join Date — Defaults to today's date. Can be backdated for existing employees.
  4. Enter Name — First Name and Last Name (both required).
  5. Enter Email — Must be unique. Will be used for login. Stored in lowercase.
  6. Set Password — Minimum 8 characters. The user can change this later via Password Reset.
  7. Add Phone (optional) — Employee's contact number.
  8. Select Role — Employee (default), Manager, HR, or Super Admin.
  9. Set Department & Designation (optional) — e.g., "Engineering" / "Software Engineer".
  10. Submit — The system creates the user with a profile, assigns initial leave balances, and generates an audit log entry.

Audit Logs

Immutable activity trail recording every significant action in the system for compliance, security, and accountability.

📜

System Audit Trail

Complete record of all administrative actions

SuperAdmin

Every significant system action is logged with the actor, timestamp, action type, affected entity, and before/after state comparison. Logs are retained for 7 years (2555 days) by default and support filtering by action, entity type, and date range.

FieldDescription

ActorUser who performed the action (name and email)
ActionType of action (PolicyUpdated, UserCreated, LeaveApproved, etc.)
Entity TypeWhat was affected (User, WorkPolicy, GeofencePolicy, OfficeLocation, etc.)

IP & User AgentNetwork and device information for security tracking
  1. Navigate to Audit Logs — Go to Admin → Audit Logs (SuperAdmin only).

  3. Review Entries — Each row shows timestamp, actor, action, entity, and a truncated entity ID.
